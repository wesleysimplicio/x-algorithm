## Epic — X Virality Skill

Ship a portable skill (`.claude/skills/x-virality/`) any AI agent can load to write X content tuned to the For You algorithm in this repo. Every claim source-grounded.

**Definition of done**
1. `SKILL.md` with frontmatter, playbook, output contract. ✅
2. References cover every signal/weight/filter/VF label/Grox screen, mapped to source. ✅
3. Score simulator that takes a draft → predicted weighted-score profile. ✅
4. Install paths for Claude Code, Codex, Tota Agent. ✅ (AGENTS.md + tota.skill.json generated)
5. Measurement loop: post-publish engagement → feedback. ✅
6. CI `verify-skill-refs` confirms every cited source path still exists. ✅

## Sprint 6 — Tooling & multi-agent integrations ✅

Goal: testable artifacts so the skill stays honest as the codebase evolves.

- ✅ **S6-1** `tools/score_simulator.py` — heuristic weighted-score predictor mirroring `ranking_scorer.rs::compute_weighted_score`. Pytest unit tests covering vqv gating, offset, diversity decay, OON factor, sign structure.
- ✅ **S6-2** `tools/verify_refs.py` — scans every `.md` in the skill, extracts cited paths, asserts each resolves (repo-relative, file-relative, skill-root-relative, or bare-filename anywhere). Pytest unit tests + live regression.
- ✅ **S6-3** CLI for the pre-publish checklist — `tools/checklist_cli.py`. Interactive walker + non-interactive / JSON modes. Pytest tests.
- ✅ **S6-4** Codex `AGENTS.md` variant — generated by `tools/generate_agents.py` from `SKILL.md` frontmatter. Lives at repo root.
- ✅ **S6-5** Tota Agent skill packaging — `tota.skill.json` at repo root, also generated by `tools/generate_agents.py`. Pytest tests.
- ✅ **S6-6** Post-publish measurement loop — `tools/measurement_loop.py`. Records predicted vs actual signal counts, aggregates over many posts, flags large drifts. Pytest tests.
- ✅ **S6-7** CI workflow that runs `verify_refs` + `pytest` + `generate_agents --check` on every PR — `.github/workflows/verify-skill.yml`.

## Out of scope (do not promote into a sprint)

- Building a Phoenix model clone. The simulator is heuristic and explicit about it.
- Tracking runtime parameter numeric values. Weights are runtime configs — teach structure, not numbers.
- X Ads campaign optimization (separate playbook).
- Automation / bots (against ToS).
- Off-platform amplification.
